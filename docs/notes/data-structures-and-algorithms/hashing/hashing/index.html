Hashing Scheme

Static

Linear Probing

Resolve collision by linearly searching and insert in next place Find: Iterate till empty slots Delete: Tombstones, you could also compact but that get’s tricky

Robin Hood Hashing

Variant of Linear Probing, take from rich and give to poor Rich = someone who is at it’s place Poor = someone who is away from it’s place

Achieved by keeping an offset of position

Cuckoo Hashing

Use multiple hashing table with different hashing functions

Find = O(1), it should be there Insert = find a place from among the table, if both tables are full, adjust the values If infinite loop = you have to increase the size

Dynamic

Dynamically grow/reduce the table

Chained Hashing

Regular hashing by chaining, linked list that grows forever

Extendible Hashing

Use buckets like chained hashing When the bucket get’s full, split it Bucket is decided on bits So, 00, 01, 11, 10 will have 4 buckets, now, if first bucket overflows, we will double the size of hash table and consider the first 3 bits to decide the bucket. the newer bucket will point to old bucket overflowing bucket will get split

Problem Everytime the size of the table doubles

Linear Hashing

Maintain a 2 hashing functions => hash() % n and hash % 2n Based on the split point, you decide whether you want to use the first has function or second hash function

Find: Use the first has function, if index if less than split value and value not in bucket, hash using second Insert: Same as find, use first and then second

Everytime an overflow happens, we increase the split pointer and split the bucket (not the one that overflew but the one at split location) Soon, we will reach the end of array, discard the first hash function, move the pointer to top, and keep doint it.
