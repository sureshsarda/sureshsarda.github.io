Concurrency Vs Parallelism

  • Concurrecny - running more than one thing together on a single resource, interleaved
  • Parallelism - you have more than one resource to run

Synchronization

  • synchronized keyword - uses the internsic lock of the object.
  • All synchronized methods are together in an object
  • synchronized keyword with static method - uses the internsic lock of the class

Re-entrant Synchronization

  • Allows to acquire a lock which you already have

Atomic Access

  • Assignment is atomic, but a++ is not
  • Operations on volatile variales are atomic? #ReadMore
  • Volatile keyword makes variable stored in main memory that CPU cache, since every CPU will have a cache, updating the value may just update the value in the cache and not in main memory. Volatile forces it to store in main memory.
