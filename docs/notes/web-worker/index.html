Service Workers

  • Runs in background seperate from the web page
  • Push Notification, background sync
  • Intercept, handle network requests
  • Can’t access dom
  • WebSockets not supported

Registering a service worker

if ('serviceWorker' in navigator) {
    window.addEventListener('load', function () {
        navigator.serviceWorker.register('/sw.js').then(function (registration) {
            // Registration was successful
            console.log('ServiceWorker registration successful with scope: ', registration.scope);
        }, function (err) {
            // registration failed :(
            console.log('ServiceWorker registration failed: ', err);
        });
    });
}

You can register as many times you want, browser will keep it once.

Installing new one

  • When you update the JS file, browser will consider it as new.
  • It will wait for all active pages to close and will keep the new worker in waiting state
  • Once everything closed - worker will reach activated

Shared Workers

  • A worker that can be accessed from several browsing scopes
  • Must share same origin - host, port, protocol

Create a new one

var myWorker = new SharedWorker('worker.js');
myWorker.port.start();

Communication

first.onchange = function () {
    myWorker.port.postMessage([first.value, second.value]);
    console.log('Message posted to worker');
}

second.onchange = function () {
    myWorker.port.postMessage([first.value, second.value]);
    console.log('Message posted to worker');
}

myWorker.port.onmessage = function (e) {
    result1.textContent = e.data;
    console.log('Message received from worker');
}

Notes

  • Not supported in IE and Safari
  • Many mobile browsers don’t support it
