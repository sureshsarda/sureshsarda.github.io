Federation

The database is partitioned by functional use cases. For example, you will end up with databases like CustomerDb, ProductsDb, etc

Sharding

The database is partitions the data bases on the value of each object or a key in an object. For example, the UserDb itself will be stored on multiple nodes based on (perhaps) the name.

Consistent Hashing

A hashing technique that doesn’t care about the number of buckets that you have. It hashes to an abstract value which then is kept in certain bucket.

Benchmark

Simulate high load on the system to understand the capacity of the system

Profile

Track performance of applications, track performance issues

BASE

BASE are typically described as properties of a NoSQL system.

  • Basically Available - the system has availability gurantee
  • Soft State - the state of system may change even without input
  • Eventual Consistency - the system will attain consistency over a period of time

ACID

ACID properties are typically of a SQL like system

  • Atomicity
  • Consistency
  • Isolation
  • Durability

Key-Value Store

A hash-table like database. A database that stores value or values against a given key. Implementations: Apache Cassandra, Redis

Columnar or Column-wise Database

Database or storage engines that store data related to columns together unlike typical row based database where an entire row is stored together. Column based databases are typically used for data analytics when you need to run query on large amount of data. These databases reduce the number of disk operations in certain uses cases. Example: AWS Redshift

Row wise database

Databases that store data of a row together. Typical SQL databases follow this approach. Retrival of an entire row is faster but aggregating the data may need multiple disk reads. Example: MySQL

Document Store

Like Key-value, but instead of a value of values, an entire document is stored. Example: MongoDb, which stores Json documents, CouchDb, DynomoDb are other examples
