Terms

Wide Column

Aka Extensible Record Stores, can hold very large number of columns or attributes for a key (can be in billions). Tyically schema free and with chracteristic of Document Stores

These should not be confused with column oriented data bases which store the data in columnar format for certain kind of workload (OLAP). Columnar = records are stored not one after the other but values of columns are stored together.

Examples: Cassandra, HBase, AzureCosmosDb

Column Oriented Database or Columnar Stores

Instead of storing the data record by record one after the other, these databases keep the data of a single column across records together. This helps in faster aggregation of data in OLAP workloads unline record store which needs to do multiple disk scans to get the required values.

Document Stores

Imagine you are storing a JSON object with a key. No schema, a column can have different data type. MongoDb is the classic example but Amazon DynamoDb, Couchbase, Azure Cosmos Db are few more.

These databases are inspired for Google Big Table. Link to paper
